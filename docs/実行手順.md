### 🔧 方法2: ステップバイステップ（推奨）

#### ステップ1: GitHubトークンの取得

1. 以下のURLを開く：  
   👉 **https://github.com/settings/tokens**

2. **"Generate new token (classic)"** をクリック

3. 以下を設定：
   - **Note**: `Portfolio Automation`
   - **Expiration**: `90 days`
   - **Select scopes**:
     - ✅ **repo** （全サブ項目）
     - ✅ **workflow**
     - ✅ **write:packages**

4. **"Generate token"** をクリック

5. ⚠️ **重要**: トークンをコピー（`ghp_`で始まる文字列）

#### ステップ2: トークンの設定

```bash
cd /home/user/webapp
python setup_token.py
```

プロンプトが表示されたら、コピーしたトークンを貼り付けてください。

#### ステップ3: リポジトリの作成

```bash
cd /home/user/webapp
python create_repositories.py
```

スクリプトが自動的に以下のリポジトリを作成します：

1. ✅ **jinno-ai** - プロフィールREADME
2. ✅ **enterprise-rag-system** - RAGシステム
3. ✅ **llm-agent-framework** - エージェントフレームワーク
4. ✅ **realtime-edge-detection** - リアルタイム物体検出
5. ✅ **multilingual-sentiment-analyzer** - 多言語感情分析

## 🔒 セキュリティ注意事項

### ⚠️ トークンを安全に保つ：

1. **絶対に公開しない**
   - `.env`ファイルは`.gitignore`に含まれています
   - スクリーンショットに含めない
   - SNSやチャットに貼り付けない

2. **トークンが漏洩した場合**
   - すぐに https://github.com/settings/tokens でトークンを削除
   - 新しいトークンを作成

3. **定期的な更新**
   - 90日後に有効期限が切れます
   - その時に新しいトークンを作成してください

## 🆘 トラブルシューティング

### ❌ "GITHUB_TOKEN not found"エラー

**解決策:**
```bash
cd /home/user/webapp
python setup_token.py
```
または、手動で`.env`ファイルを作成してください。

### ❌ "Repository already exists"エラー

**解決策:**  
既存のリポジトリを検出すると、更新するか確認されます。  
`y`を入力すると既存のREADMEを更新します。

### ❌ "Authentication failed"エラー

**解決策:**
1. トークンが正しくコピーされているか確認
2. 必要な権限（repo、workflow）が選択されているか確認
3. トークンの有効期限が切れていないか確認

### ❌ PyGithubがインストールできない

**解決策:**
```bash
pip install --upgrade pip
pip install PyGithub python-dotenv
```
